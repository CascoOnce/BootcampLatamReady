SESIÓN 6: Introducción de Git y GitHub

Esta es una prueba sobre como manejar los versionados en Git

Primero git add.

git config pull.rebase true => Mis cambios son los reales.

git config pull.rebase false => Hay que decidir

git reset = moverme entre commits

git branch = mostrar las ramas

git checkout -b "" = agregar una nueva rama
